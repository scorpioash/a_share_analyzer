覆盖 **行情 · 资金 · 板块 · 基本面 · 资讯 · 情绪** 六大维度，辅以 AI 深度推演，为你打造一站式 A 股投研决策中枢。

## 🛰️ 实时数据源 · 多源降级机制

本工具采用**多数据源冗余设计**，即使单一接口失败也能保证实时分析正常运行：

| 优先级 | 数据源 | 用途 | 延迟 | 代理友好度 |
|------|------|------|------|------|
| P0 | 东方财富（直连实现） | 实时行情、分时 | 秒级 | ⚠️ 挂代理易失败 |
| P1 | 雪球 | 个股实时快照 | ~1 秒 | ✅ 稳定 |
| P2 | 新浪 | 全市场快照、日内分时 | ~30 秒 | ✅ 稳定 |
| P3 | 腾讯 | 日线兜底、昨收补位 | 收盘级 | ✅ 稳定 |

**任何一个成功就算成功**，系统会自动融合多源数据取最优。诊断面板里可一键测试所有数据源状态。

> **💡 模型选择建议**：金融市场犹如没有硝烟的战场，盘面研判极其考验 AI 的逻辑推理深度。为了让本系统输出最敏锐、最准确的操作建议，**强烈建议搭配使用 Gemini-3.1、Claude-4.6 或 ChatGPT-5.4 等顶级推理模型**。

## 🚀 从零开始部署（保姆级教程）

### 第一步：安装 Python 环境

本项目需要 Python 3.8 或更高版本。如果你的电脑上还没有安装 Python，请按以下步骤操作：

**Windows 用户：**
1. 打开浏览器，访问 [Python 官方下载页](https://www.python.org/downloads/)
2. 点击黄色的 **"Download Python 3.x.x"** 按钮下载安装包
3. 双击运行安装包，**⚠️ 一定要勾选底部的 "Add Python to PATH"（把 Python 添加到环境变量）**
4. 然后一路点 "Next"、"Install" 直到安装完成

**Mac 用户：**
1. 打开「终端」应用（在启动台搜索"终端"即可找到）
2. 输入以下命令安装 Homebrew（如果已安装可跳过）：
   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```
3. 然后输入：
   ```bash
   brew install python
   ```

**如何验证安装成功？**
- Windows：按 `Win + R`，输入 `cmd` 回车，在黑框中输入 `python --version`
- Mac：打开终端，输入 `python3 --version`
- 如果显示 `Python 3.x.x` 就说明安装成功了

---

### 第二步：下载本项目

**方法 A（推荐新手）：直接下载压缩包**
1. 在本项目的 GitHub 页面，点击绿色的 **"Code"** 按钮
2. 选择 **"Download ZIP"**
3. 下载完成后，右键解压到你喜欢的目录（比如桌面）

**方法 B：使用 Git 克隆**
```bash
git clone <本项目的 GitHub 地址>
```

---

### 第三步：一键启动！

**Windows 用户：**
1. 打开解压后的项目文件夹
2. 找到 `start.bat` 文件，**双击运行**
3. 第一次运行会自动安装所有依赖（需要等待 1-2 分钟，期间不要关闭黑框窗口）
4. 如果黑框中出现 `Email:` 提示，**直接按回车键跳过**即可
5. 安装完成后，浏览器会自动打开 `http://localhost:8501`

**Mac / Linux 用户：**
1. 打开终端，`cd` 进入项目目录
2. 执行以下命令：
   ```bash
   chmod +x start.sh
   ./start.sh
   ```
3. 同样等待依赖安装完成，浏览器会自动打开

---

### 第四步：配置你的 AI 模型

启动成功后，你会看到网页界面。接下来需要配置一个 AI 模型才能使用诊股功能：

1. 点击左侧边栏的 **"🔑 快捷配置 API"** 展开配置面板
2. 在 **"选择驱动引擎"** 中选择你的模型厂商：
   - **ChatGPT (OpenAI)**：需要 [OpenAI 官网](https://platform.openai.com/api-keys) 注册并生成 API Key
   - **Claude (Anthropic)**：需要 [Anthropic 官网](https://console.anthropic.com/) 注册并获取 Key
   - **Gemini (Google)**：需要 [Google AI Studio](https://aistudio.google.com/apikey) 获取 Key
   - **自定义 (OpenAI 协议)**：适用于 DeepSeek、OneAPI 中转站等所有兼容 OpenAI 格式的服务商。需要手动填入 API URL 和模型名
3. 填入 **API KEY** 和 **模型名**
4. 点击 **"保存配置并重载"**
5. 配置完成！现在可以前往「智能诊股」或「板块分析」页面开始使用了

## 🌐 网络与代理配置（挂代理用户必看）

如果你使用 Clash / V2rayN / Surge 等代理客户端（无论是全局模式、规则模式还是 TUN 模式），**可能会遇到东方财富接口连接失败**，报错类似 `ProxyError: Max retries exceeded`。

### 为什么会出现这个问题？

你的代理客户端默认会把国内网站判为"直连"，但：
1. 代理客户端把请求交给"直连"处理
2. 直连出站时，某些网络环境（公司防火墙、ISP、特定 CDN 黑名单）会拦截
3. 于是东财接口连接被重置，报 `ProxyError`

### 这会影响使用吗？

**几乎不影响**。本工具有多源降级机制：东财访问不了时会自动切到**雪球**（秒级实时）和**新浪**（30 秒延迟），依然能拿到今日真实行情。

### 如果你想解决东财问题（三种方案）

#### 方案 1：切换网络模式（代码层）

在 `.env` 文件里加一行：

```bash
# 默认 system（跟随系统代理），挂代理时最合适
AK_NET_MODE=system

# 不挂代理的用户用这个
# AK_NET_MODE=bypass

# 挂代理但 PAC 规则把东财误判为直连时用这个，强制走指定代理
# AK_NET_MODE=force
# AK_PROXY_URL=http://127.0.0.1:7890
```


#### 方案 2：不折腾，直接用（推荐）

什么都不改，让系统走雪球+新浪路线。延迟 1-30 秒，对 T+3 级别的分析完全够用。

### 诊断面板怎么用？

遇到数据抓取问题时，在「智能诊股」页面展开 **"🔍 实时数据抓取诊断"** 面板：

- **⚡ 测试东财直连 API**：单独测试东财是否可达
- **🔍 探测本地 Clash/Mihomo**：生成 Clash 配置片段
- **🧪 测试所有数据源**：一次性检查雪球/新浪/腾讯/东财所有通道
- **网络模式切换**：UI 上实时切换 `system` / `bypass` / `force` 三种模式

## 🤖 GitHub Actions 自动推送（无需服务器）

如果你希望每天收盘后自动分析并推送到飞书/微信：

1. Fork 本仓库到你的 GitHub 账号
2. 进入你 Fork 后的仓库 → `Settings` → `Secrets and variables` → `Actions`
3. 添加以下配置变量：
   - `OPENAI_API_KEY`：你的模型 API Key
   - `FEISHU_WEBHOOK_URL`：飞书机器人的 Webhook 地址
   - `STOCK_LIST`：想跟踪的股票代码，如 `600519`
4. 搞定！每个交易日下午 16:00 会自动运行诊断并推送结果

## ❓ 常见问题 FAQ

<details>
<summary><b>Q1：诊股报告显示"实时数据不可得"怎么办？</b></summary>

打开诊断面板点击 **"🧪 测试所有数据源"**，查看哪一层挂了：

- **雪球和新浪都挂** → 检查你的网络是否能访问 `xueqiu.com` / `sina.com.cn`
- **只有东财挂** → 正常现象，系统会自动用雪球+新浪，数据一样可靠
- **所有源都挂** → 网络不通，检查防火墙或 DNS
</details>

<details>
<summary><b>Q2：分析出来的价格跟行情软件不一样？</b></summary>

正常现象，可能原因：
- **雪球数据延迟 1 秒，新浪延迟 30 秒** —— 盘中高频波动时会略有差异
- **东财挂时用的是腾讯日线** —— 这是昨收数据，不是今日实时（诊断面板会标注数据源）

确认方法：看报告 `## 0.` 节的 `数据源` 字段，`EM-Direct`/`XueQiu`=实时，`TX-Daily(stale)`=昨日。
</details>

<details>
<summary><b>Q3：AI 输出了错误的昨收/今收价格？</b></summary>

本工具已经做了抗数据幻觉设计，所有价位（阻力/支撑/均线）都是**基于数据包实时计算**而非 AI 记忆。如果仍出现错误，请检查：

1. 诊断面板里数据源是否为"今日" (`今日=True`)
2. 如果数据源是 `TX-Daily(stale)` 则说明实时源全挂，报告里会明确标注"基于最近一个交易日数据"
</details>

<details>
<summary><b>Q4：Gemini 3.1 / Claude 4.7 报错"模型不存在"？</b></summary>

检查 `.env` 里的模型名是否正确：
- Gemini：`gemini-3.1-pro-preview`
- Claude：`claude-sonnet-4-6` 或 `claude-opus-4-7`
- OpenAI：`gpt-4o` / `gpt-5.4`（取决于你的 API 账号权限）
</details>

<details>
<summary><b>Q5：策略文件 my_strategy.md 怎么写效果最好？</b></summary>

建议包含以下几块：
- **铁律条款**：写死的纪律（比如"偏离 MA5 超 5% 严禁追高"）
- **评级词表**：限定 AI 可用的评级词（避免它自己造词）
- **历史经验**：你总结的买点/卖点信号特征
- **仓位管理**：不同评级对应的建议仓位

越具体越好，AI 会严格按照你的策略输出。
</details>
